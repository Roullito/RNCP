## Module 3 – Dossier Professionnel (DP)

### 3.1. Rôle du Dossier Professionnel

Le **Dossier Professionnel (DP)** est un document **distinct** du dossier de projet :

- Le **dossier de projet** présente un ou plusieurs projets techniques en détail (dont ton Smart Trading Bot).
- Le **dossier professionnel** présente **ton parcours**, tes **expériences** et la manière dont tu as **mis en œuvre les compétences du titre** dans des situations réelles ou de formation.

Le DP sert de **support principal pour l’entretien final** avec le jury.  
Il permet de vérifier que tu maîtrises les **activités types** du titre RNCP, au-delà d’un seul projet.

> ⚠️ Il doit comporter **au moins 1 exemple par activité type** du titre (pour DWWM : AT1 front, AT2 back).

---

### 3.2. Différence entre dossier de projet et dossier professionnel

| Aspect                      | Dossier de projet                                         | Dossier professionnel                                           |
|----------------------------|-----------------------------------------------------------|-----------------------------------------------------------------|
| Finalité                   | Présenter un ou plusieurs projets techniques              | Présenter ton parcours pro / formation et tes compétences       |
| Focalisation               | Un projet détaillé (cahier des charges, env. technique…) | Plusieurs expériences/situations (projets, stages, jobs, etc.) |
| Support principal pour     | **Présentation de projet** + entretien technique          | **Entretien final**                                             |
| Structure                  | Plan imposé par le REV (compétences, besoins, env., réalisations) | Plan du template officiel DP (activités types, situations, déclaration sur l’honneur) |
| Niveau de détail technique | Très détaillé sur 1 projet                               | Synthétique mais orienté **compétences + responsabilités**     |

Les deux documents sont **complémentaires** :  
- Le dossier de projet montre comment tu as **conduit un projet**.  
- Le DP montre comment tu t’inscris dans le **métier de développeur** au sens large (activités types).

---

### 3.3. Objectif du DP pour le RNCP5 DWWM

Pour le titre DWWM, les **activités types** sont, de manière simplifiée :

- **AT1** – Développer la partie front-end d’une application web ou web mobile sécurisée  
- **AT2** – Développer la partie back-end d’une application web ou web mobile sécurisée  

Le DP doit :

1. Présenter **au moins une situation** (projet / expérience) qui illustre **AT1**.  
2. Présenter **au moins une situation** (projet / expérience) qui illustre **AT2**.  
3. Mettre en avant les **compétences professionnelles** du REAC (CP1 → CP8).  
4. Être **cohérent avec ton parcours et ton GitHub** (les projets évoqués doivent être visibles et crédibles).
5. Contenir une **déclaration sur l’honneur** dûment complétée et signée.

---

### 3.4. Plan type d’un Dossier Professionnel DWWM

Le template officiel Holberton / Ministère ne doit pas être modifié (tu dois en faire une copie et le remplir).  
Mais on peut résumer sa logique de contenu comme suit :

1. **Informations générales**
   - Identité, coordonnées
   - Titre visé (DWWM) et organisme de formation
   - Situation actuelle (formation, recherche d’emploi, alternance…)

2. **Parcours de formation**
   - Holberton School : période, contenu global, principaux langages et projets
   - Autres formations / certifications éventuelles

3. **Parcours professionnel**
   - Emplois, stages, missions d’extra (même hors dev) : ce que tu y as appris (organisation, rigueur, travail en équipe, gestion du stress, etc.)

4. **Description des activités types du titre**
   - Pour chaque activité type (AT1, AT2) :
     - Choisir **au moins une “situation de travail”** (un projet ou une expérience concrète).
     - Décrire cette situation selon un canevas structuré :
       - Contexte (où, quand, avec qui ?)
       - Objectifs (quoi livrer ? quel résultat attendu ?)
       - Moyens (outils, technologies, organisation)
       - Ton rôle (ce que **toi** tu as fait)
       - Résultats (ce qui a été obtenu, ce qui fonctionne)
       - Compétences mobilisées (liens explicites avec le REAC)

5. **Synthèse des compétences**
   - Tableau ou paragraphe qui montre :
     - Quelles compétences professionnelles du DWWM tu as mises en œuvre
     - Dans quelles situations / projets

6. **Déclaration sur l’honneur**
   - Phrase type indiquant que tu certifies l’exactitude des informations
   - Date, signature

### 3.5. Application à mon parcours (brouillon de choix des situations)

Pour ton cas, un brouillon réaliste pourrait être :

- **Activité type 1 (AT1 – Front-end)**  
  - Exemple 1 : **Portfolio développeur / site personnel**  
    - Création d’une interface web responsive  
    - Intégration de maquettes (Figma ou équivalent)  
    - Utilisation de HTML/CSS/JS, éventuellement un framework CSS  
  - Exemple 2 (option) : **Dashboard web pour le Smart Trading Bot**  
    - Pages permettant de sélectionner l’actif et la stratégie  
    - Affichage des résultats (graphiques, listes de trades, indicateurs)

- **Activité type 2 (AT2 – Back-end)**  
  - Exemple 1 : **Smart Trading Bot**  
    - Récupération de données via API (Bitget)  
    - Stockage des données (BDD, fichiers structurés)  
    - Composants métier (indicateurs, signaux, exécution de stratégies)  
    - Gestion de la sécurité (clé API, validation, logs)
  - Exemple 2 (option) : **Projet de type HBnB / appli de réservation / projet Flask/Django**  
    - API REST, accès BDD, gestion des utilisateurs, etc.

- **Expériences complémentaires possibles** (dans le DP, pour renforcer l’image globale) :
  - Projets Holberton significatifs (HBnB, ORM, REST API…)
  - Missions d’extra en restauration / hôtellerie :
    - Travail en équipe
    - Gestion des pics d’activité, rigueur, fiabilité
    - Relation client

### 3.6. Méthode pour remplir concrètement le DP

1. **Récupérer le bon template**  
   - Copier le fichier :  
     - `DOSSIER PROFESSIONNEL (DP) RNCP 37674 DWWM.docx`  
   - Ne pas modifier la structure du document (titres, ordre des rubriques).

2. **Lister d’abord les situations possibles**
   - Activité type 1 : choisir 1 à 2 projets très orientés front (portfolio, dashboard bot…).  
   - Activité type 2 : choisir 1 à 2 projets très orientés back (Smart Trading Bot, HBnB, API…).  

3. **Pour chaque situation choisie, rédiger au brouillon :**
   - Contexte (où / quand / avec qui ?)
   - Objectifs (attendus techniques et fonctionnels)
   - Moyens (technos, organisation, outils)
   - Ton rôle précis (ce que tu as fait toi, pas “on”)
   - Résultats (ce qui fonctionne, ce qui a été livré)
   - Compétences mobilisées (lien avec les activités types / compétences REAC)

4. **Transférer ces textes dans le template DP**
   - Adapter la longueur : être complet mais synthétique
   - Garder un style professionnel, clair, sans jargon inutile

5. **Compléter la partie “Parcours”**
   - Formations : Holberton + autres
   - Expériences professionnelles : jobs d’avant, extras, missions ponctuelles

6. **Finaliser et vérifier**
   - Compléter la **déclaration sur l’honneur** (obligatoire)
   - Relire avec l’idée : “Est-ce que le jury comprend ce que je sais FAIRE dans le métier ?”
   - Exporter en **PDF**
   - Transmettre à l’équipe campus dans les délais indiqués
